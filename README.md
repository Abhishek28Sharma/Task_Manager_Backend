### ­¤¦¬ Example Requests

## 1. Get All Tasks

Retrieve a list of every task stored in the database.

```
Method: GET
Endpoint: /tasks
Request: No body required.
Backend Logic: Calls Task.find() to fetch all documents.
```

Response:

- **Success (200 OK)**: An array of task objects containing \_id, title, description, completed, and createdAt.

- **Error (500)**: JSON object with an error message if the database fetch fails.

---

### 2. Create New Task

---

Add a new task to the manager.

Method: POST
Endpoint: `/tasks`

Request Body:

```json
{
  "title": "String (Required)",
  "description": "String (Optional)"
}
```

- **Backend Logic:** Validates input, creates a new Task instance, and saves it to MongoDB.

Response:

- **Success (201 Created):** The newly created task object including its generated \_id and createdAt timestamp.

- **Error (500):** JSON object describing the validation or server error.

---

### 3. Update Task (Edit or Toggle Status)

Modify an existing task's details or mark it as completed/incomplete.

- Method: `PATCH`

- Endpoint: `/tasks/:id`

- URL Parameter: `id` ŌĆö The unique MongoDB` _id` of the task.

- Request Body:

```json
{
  "title": "String (Optional)",
  "description": "String (Optional)",
  "completed": "Boolean (Optional)"
}
```

- Backend Logic: Finds the task by ID and applies the updates using findByIdAndUpdate. It returns the document after updates are applied.

Response:

- **Success (200 OK):** The updated task object.

- **Error (404):** "Task not found" if the ID does not exist.

- **Error (500):** Server error message.

---

### 4. Delete Task

Permanently remove a task from the system.

- Method: `DELETE`

- Endpoint: `/tasks/:id`

- URL Parameter: id ŌĆö The unique MongoDB `_id` of the task.

- Request: No body required.

- Backend Logic: Calls `findByIdAndDelete` using the provided ID parameter.

Response

- **Success (200 OK):**

```json
{"message": "Task deleted successfully"}.
```

- **Error (404):** "Task not found" if the ID does not exist.

- **Error (500):** Server error message.
